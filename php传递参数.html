php参数传递(PHP表单数据接收)

来源： 作者： 时间：2007-10-25 Tag： 点击： 1677
问个问题:关于PHP里面的参数传输问题:
在mainc.php中用到表单:
<form id="form1" name="form1" method="post" action="form4.php">
其中定义了:
<input name="name" type="text" size="18" maxlength="5" height="20" />
<input type="submit" name="Submit" value="提交" />
<input type="reset" name="Submit2" value="重置" />
<input type="hidden" name=send1 value="send">
但是在form4.php中,想要
<? if($send1=="send")
{ echo("您的基本信息如下:");
print("姓名:" .$name.);}?>
却不能很好地将name参数传递过来呢?究竟要怎么做才能传过来咧?
若能在一天之内回答好.追加13分.
问题补充：就像第一个回答者,的确是解决了问题,但是我在看书的时候,并没有见过需要加那么多的GET的呀?是书的问题,还是PHP配置上有问题,还是怎么回事呢?我把mainc.php和form4.php放在同一文件夹里

PHP4.1（好像）以后取消了直接接收变量，而用 $_GET['变量名'] 接收从链接传递过来的变量， 用 $_POST['变量名'] 接收从表单提交来的变量。同样有$_FILES[],$_SERVER[]等，你看的书介绍的是以前版本的php吧。
建议适应新的写法，或者:
把php.ini文件中，register_globals设为on。(不建议)

php4中网页间传递参数
在接收参数的网页中一定要用$_GET接受
在form4.php中，$name=$_GET['name'];
然后就可以随心所欲的运用$name变量喽

if($_GET['send1']=="send")

print("姓名:" .$_GET['name'].);}?>
这样吧。。。改这两个地方。
--------------------
即然你问了我就再说一下，不是书的问题，而是PHP版本问题，PHP以前是那样用，不过后来就需要这个变量了，你以后看书最好找PHP5的书看。
